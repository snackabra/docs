Class: SBCrypto#

Utility class for SB crypto functions. Generally we use an object instantiation of this (typically ‘’sbCrypto’’) as a global variable.

generateIdKey#

▸ generateIdKey(buf): Promise<{ idBinary: ArrayBuffer ; keyMaterial: ArrayBuffer }>

Hashes and splits into two (h1 and h1) signature of data, h1 is used to request (salt, iv) pair and then h2 is used for encryption (h2, salt, iv).

wrap#

▸ wrap(body, sender, encryptionKey, salt, signingKey): Promise<ChannelMessage>

Internally this is Deprecated, but we retain a simplified version for now; for example, some unit tests use this to ‘track’ higher-level jslib primitives. This used to be the main approach to boot-strap a ChannelMessage object; this is now divided into sync and async phases over internal channel queues.
